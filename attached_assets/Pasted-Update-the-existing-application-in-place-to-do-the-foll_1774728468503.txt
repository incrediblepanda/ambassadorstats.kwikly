PART 1: AMBASSADOR BACKEND API — FINAL INTEGRATION RULES
==================================================

Use the Ambassador Backend API only.

Do not use:
- the Ambassador JS API
- browser snippets
- frontend widget integrations
- client-side Ambassador embed logic

This app is a server-side admin platform and must use the Backend API only.

Official implementation details:
- Base URL: https://api.getambassador.com/api/v2
- Auth method: username and token must be included as URL path segments on every request
- Full request URL pattern:
  https://api.getambassador.com/api/v2/{username}/{token}/json/{endpoint}/
- POST requests must use:
  Content-Type: application/json
- TLS 1.2+ is required
- Standard API rate limits exist
- No documented IP allowlisting requirement
- Replit should be able to call the API directly from the server side

Important:
Do not implement Basic Auth.
Do not implement Bearer token headers.
Do not implement custom auth headers unless explicitly required by a specific documented endpoint.
The default auth pattern for this integration is URL-path-based auth using:
{username}/{token}/json/
