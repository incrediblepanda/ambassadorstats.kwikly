Build a full-stack admin platform for managing ambassador referral dashboards.

Project goal:
Create an admin-only web application that syncs ambassador and referral performance data from GetAmbassador, stores it in the Replit database, and generates unique iframe dashboard links for each ambassador. Ambassadors will not log into this platform directly. Only internal admins will log in to manage integrations, review synced ambassador records, and access the generated iframe links used inside GetAmbassador.

Core business context:
- GetAmbassador is the source of truth for ambassador and referral dashboard data
- HubSpot feeds data into GetAmbassador via webhook, but this app should treat GetAmbassador as the source of truth
- Ambassadors refer two lead types:
  1. Professional / temp worker leads
  2. Company / office leads
- Attribution is based on the ambassador short code
- Each ambassador needs a unique dashboard link that will be embedded as an iframe inside GetAmbassador
- Ambassadors will never log into this admin platform
- Only internal admins need access to this application
- The system must automatically create synced ambassador dashboard records whenever a new ambassador is created in GetAmbassador

Platform requirements:
This is an admin-only operations platform, not a multi-user customer app.

Admins need to be able to:
- log into the admin platform
- manage GetAmbassador connection settings
- view sync status
- manually trigger sync jobs
- see all synced ambassadors
- click into an ambassador record
- view the ambassador’s generated iframe dashboard link
- copy the iframe dashboard link for placement into GetAmbassador
- see whether an ambassador dashboard account has been auto-created successfully

System behavior:
- Whenever a new ambassador is detected from GetAmbassador, automatically create a synced ambassador account record in the platform database
- Automatically generate:
  - dashboard slug
  - secure dashboard token
  - full iframe URL
- Persist all ambassador dashboard metadata in the database
- Maintain sync status and timestamps for each ambassador record

Technical stack:
- Build this for Replit
- Use Replit’s database as the main persistence layer
- Use a modern JavaScript or TypeScript full-stack architecture
- Preferred stack:
  - Frontend: React
  - Backend: Node.js + Express
  - Styling: Tailwind CSS
- Organize the project clearly for future expansion
- Use environment variables for secrets and API credentials

Authentication requirements:
- Admin-only authentication
- Do not build ambassador login
- Only a small number of internal admins should be able to access the platform
- Implement a simple and secure admin auth system suitable for internal use
- Support login via email/password stored securely, or a lightweight admin auth pattern appropriate for Replit
- Protect all admin pages and admin API routes
- The public iframe dashboard route should not require admin login, but must require a secure per-ambassador token
